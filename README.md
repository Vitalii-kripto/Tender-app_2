# TenderSmart Gidroizol AI 🏗️🤖

Интеллектуальная система автоматизации тендерных продаж строительных и гидроизоляционных материалов. Приложение использует передовые возможности искусственного интеллекта (Google Gemini 2.5 Flash) для анализа тендерной документации, точного подбора аналогов продукции и минимизации юридических рисков.

## 🌟 Основные возможности

### 1. Умный подбор аналогов и "Тройная проверка" (Product Matching)
Сердце системы — интеллектуальный модуль работы с техническими заданиями (ТЗ) и коммерческими предложениями (КП):
*   **Поиск аналогов:** Автоматический подбор материалов из внутренней базы (каталога) или глобальной сети на основе сложных технических требований.
*   **Тройная проверка (ТЗ ↔ КП ↔ Интернет):** ИИ анализирует требования заказчика, предложенные вами материалы и *реальные* технические листы (TDS) из интернета (с использованием Google Search Grounding). Система выявляет несоответствия, завышенные характеристики и откровенный обман (статус `FAKE`).
*   **Волшебная палочка (Enrichment):** Автоматическое обогащение характеристик товара реальными данными из сети в один клик.

### 2. Анализ юридических и финансовых рисков (Legal Analysis)
*   Загрузка тендерной документации (PDF, DOCX и др.) для автоматического анализа.
*   ИИ выявляет скрытые угрозы: жесткие штрафные санкции, невыполнимые сроки поставки, кабальные условия оплаты, требования избыточных или редких сертификатов.
*   Генерация сводного отчета с общим уровнем риска (Низкий, Средний, Высокий) и конкретными рекомендациями для тендерных специалистов и юристов.

### 3. Валидация пакета документов (Compliance Check)
*   Проверка готовности заявки перед подачей на ЭТП (электронную торговую площадку).
*   Система проверяет загруженный пакет файлов на наличие всех обязательных форм (Форма 2 / Конкретные показатели, анкеты участника, декларации принадлежности к СМП и т.д.) и базовую корректность их заполнения.

### 4. CRM для тендерных продаж
*   Единое рабочее пространство для ведения тендеров по этапам воронки продаж (Новый → Анализ → Подготовка → Подано → Выиграно → Проиграно).
*   Глубокая интеграция с модулями анализа: данные выбранного тендера автоматически подтягиваются в инструменты ИИ для проверки.

### 5. Дашборд и аналитика
*   Наглядное отображение ключевых метрик: Win Rate (процент побед), общая сумма активных тендеров, количество тендеров в работе.
*   Графики активности и воронка продаж.

## 🛠 Технологический стек

*   **Frontend:** React 18, TypeScript, Vite, Tailwind CSS, Lucide Icons (для иконок), Recharts (для графиков).
*   **Backend:** Python, FastAPI, SQLAlchemy (для работы с БД).
*   **AI Engine:** Google Gemini 2.5 Flash.
*   **Интеграции ИИ:** Использование инструмента Google Search Grounding для поиска актуальных характеристик строительных материалов в реальном времени.

## 🚀 Сценарий использования (Workflow)

1.  **Поиск и добавление:** Менеджер находит релевантный тендер и заводит его карточку в CRM.
2.  **Анализ рисков:** В разделе "Анализ документации" загружается проект договора и информационная карта. ИИ подсвечивает опасные пункты. Если риски приемлемы, тендер переводится на этап "Подготовка".
3.  **Подбор материалов:** В разделе "Подбор и Валидация" загружается ТЗ заказчика. ИИ автоматически извлекает требуемые материалы. Менеджер подбирает аналоги из каталога или ищет их через ИИ.
4.  **Тройная проверка:** Нажатие кнопки "Сравнить" запускает глубокий анализ. ИИ проверяет, действительно ли предложенный аналог (например, "Техноэласт ЭПП") соответствует требованиям ТЗ, сверяясь с официальными данными производителя в интернете.
5.  **Контроль:** Перед подачей заявки готовый пакет документов прогоняется через "Валидацию форм", чтобы исключить риск отклонения заявки по формальным признакам (забыли приложить декларацию или не заполнили обязательное поле).
6.  **Подача:** Уверенная подача заявки на ЭТП.

## ⚙️ Структура проекта

*   `/src` — Исходный код Frontend (React).
    *   `/pages` — Страницы приложения (Dashboard, CRM, ProductMatching, Analysis и др.).
    *   `/components` — Переиспользуемые UI компоненты (Sidebar, Header).
    *   `/services` — API клиенты и сервисы для работы с ИИ (`geminiService.ts`).
*   `/backend` — Исходный код Backend (FastAPI).
    *   `main.py` — Точка входа, роуты API.
    *   `/services` — Бизнес-логика (`ai_service.py`, `document_service.py`).
    *   `models.py` — Модели базы данных.

## 📝 Примечание по режиму Демо
В приложении предусмотрен `IS_DEMO_MODE` (в файле `src/services/geminiService.ts`). При его активации приложение использует замоканые данные для демонстрации работы интерфейса без необходимости обращения к реальному бэкенду и API Gemini. Для полноценной работы с ИИ этот режим должен быть отключен, а бэкенд запущен с валидным `GEMINI_API_KEY`.
